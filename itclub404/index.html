А может, не надо?

		
	


	

		
В чем профит?

		
    
			
  • Более лаконичный код
    • 
			
    
				
    
					React + classnames
				
    
			
    
				vs
				
    
					React + Styled Components
				
    
			
    
			
    
		

		
	


	

		
    
			
  • Общий код у CSS и у JS
    • 
		

		

		
	


	

		
Еще плюсы:

		
    
			
  • Проще конфигурировать (webpack, node.js)
    • 
			
  • Изоляция стилей
    • 
			
  • Переносимый код
    • 
			
  • Code-splitting стилей при использовании SSR
    • 
		

		
Есть и недостатки:

		
    
			
  • Более сложный debug
    • 
			
  • Нет source-maps, либо нет выгрузки в CSS-файл
    •
